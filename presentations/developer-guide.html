‚öôÔ∏è 2. Create a Service

from ..models.posts import Post

async def create_post(
    title: str, 
    content: str, 
    author_id: str
) -> Post:
    post = Post(
        title=title,
        content=content,
        author_id=author_id
    )

üåê 3. Create Routes

# frontend/routes/posts.py
from fastapi import APIRouter
from ..deps import get_current_user
from ...services.posts import create_post

router = APIRouter(prefix="/posts")

@router.post("/")
async def create_post_route(
    title: str,
    content: str,
    user = Depends(get_current_user)
):
    return await create_post(
        title, content, str(user.id)
    )

üîê Authentication System

üåê OAuth Providers

Pre-configured OAuth providers:

  • G Google: OAuth 2.0 integration
  • GH GitHub: Developer-friendly auth
  • MS Microsoft: Enterprise integration

OAuth Flow:

1. User clicks "Login with Google"
2. Redirect to OAuth provider
3. User authenticates
4. Callback with authorization code
5. Exchange for access token
6. Create/link user account
7. Set session cookie

‚úâÔ∏è Magic Link Authentication

Passwordless authentication via email:

  • Enter email address
  • Receive secure login link
  • Click link to authenticate
  • Automatic account creation
# Magic link flow
POST /auth/magic-link
{
  "email": "user@example.com"
}

# Email sent with secure token
# User clicks: /auth/verify/{token}
# Session created, user logged in

from ..deps import get_current_user
from ...models.core.user import User

@router.get("/profile")

@router.post("/posts")

    title: str,
    content: str,
    user: User = Depends(get_current_user)

üåé Adding New Languages

Step-by-step:
  1. 1. just new-locale es - Create Spanish locale
  2. 2. Edit locales/es/LC_MESSAGES/messages.po
  3. 3. Add translations for all msgid entries
  4. 4. just compile-locales - Compile
  5. 5. Test in browser with ?lang=es
# Example .po entry
msgid "Welcome"
msgstr "Bienvenido"

msgid "You have %(count)d items"
msgstr "Tienes %(count)d elementos"

üìã Best Practices

üêç Python Code Style

  • Type hints: Always use type hints for function parameters and returns
    async def get_user(user_id: str) -> User | None:
  • Async/await: Use async functions for all database operations and I/O
    user = await User.find_one(User.email == email)
  • Auto-formatting: ALWAYS run ruff format . after changes
    ruff format . && ruff check --fix .
  • Naming: snake_case for functions/variables, PascalCase for classes
    def get_user_posts() -> list[Post]:

üé® Frontend Patterns

  • HTMX: Prefer server-side rendering with HTMX over complex JavaScript
    hx-post="/api/like" hx-target="#likes"
  • Tailwind: Use utility classes, avoid custom CSS
    class="btn btn-primary rounded-lg shadow-md"
  • DaisyUI: Use pre-built components when available
    class="card card-body bg-base-100"
  • Templates: Use Jinja2 inheritance, avoid duplication
    {% extends "base/skeleton.html.jinja" %}

üîí Security Best Practices

  • Never log secrets: Use secret masking in configuration
    api_key: SecretStr = Field(..., repr=False)
  • Input validation: Use Pydantic models for all inputs
    class CreatePost(BaseModel): title: str
  • Authentication: Use dependency injection for protected routes
    user: User = Depends(get_current_user)
  • Environment secrets: Use .env.local for sensitive data
    SECRET_KEY=your-secret-key-here

üö® Common Issues

Port Already in Use

Error: Address already in use (port 8000)

# Kill process using port
lsof -ti:8000 | xargs kill -9

# Or use different port
just local-dev 8080

CSS Not Loading

Styles not updating or missing

# Restart asset watcher
pkill -f "pnpm dev"
pnpm dev

# Clear browser cache
Cmd+Shift+R (Mac) or Ctrl+Shift+R

Database Connection

Cannot connect to MongoDB

# Check database URL
echo $MONGODB_URL

# Test connection
curl $MONGODB_URL

# Restart Docker if using containers
just dev
